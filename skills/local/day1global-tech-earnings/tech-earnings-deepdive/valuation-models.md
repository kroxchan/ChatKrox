## 1. 巴菲特视角：Owner Earnings 估值法

### 思想来源

巴菲特在1986年致股东信中提出了"所有者收益"（Owner Earnings）的概念。他认为，传统的会计利润并不能反映企业主真正能拿走的钱。Owner Earnings 是巴菲特心目中最接近"企业真实赚钱能力"的指标。

### 核心公式

```
Owner Earnings = 净利润
              + 折旧与摊销（D&A，把之前花的大笔钱分摊到每年的"虚拟成本"，并非真的花了钱）
              - 维持性资本支出（Maintenance CapEx，维持现有业务正常运转必须花的钱）
              - 营运资本变动（Working Capital Changes，被"困"在应收账款和库存里的钱）
```

**通俗理解**：想象你买了一家奶茶店。净利润是账上赚了多少，但你还得考虑：设备折旧不用真花钱（加回来），但设备坏了要换（减掉维修费），进了一堆原料还没用（钱被占用了）。剩下的才是你真的能拿回家的钱。

### 搜索和计算步骤

1. **搜索关键词**：`[股票代码] net income depreciation amortization capex` 或 `[股票代码] cash flow statement`
2. **获取数据**：
   - 净利润（Net Income）：从利润表获取
   - 折旧与摊销（D&A）：现金流量表中通常会列出
   - 总资本支出（Total CapEx）：现金流量表的"investing activities"部分
   - 维持性CapEx估算：通常取总CapEx的40-60%（如果公司没有明确区分）。对于正在大规模扩张的科技公司，维持性CapEx可能只占总CapEx的20-30%。
3. **计算 Owner Earnings**
4. **用10-15倍 Owner Earnings 估算"巴菲特合理价格"**：
   - 高确定性业务（如拥有强护城河的成熟科技公司）：可用12-15倍
   - 一般确定性：用10-12倍
   - 注意：巴菲特传统上偏好低倍数，但优质科技公司可以给更高倍数

### 判断标准

| Owner Earnings 倍数 | 判断 |
|-------------------|------|
| 当前股价 < 10x OE | 明显低估（按巴菲特标准） |
| 当前股价 = 10-15x OE | 合理区间 |
| 当前股价 = 15-25x OE | 偏贵，需要高增长支撑 |
| 当前股价 > 25x OE | 高估（除非增长极为确定） |

### 巴菲特会额外关注什么

- 这家公司是否有他所说的"经济特许权"（Economic Franchise）——即不可替代的产品或服务？
- 管理层是否诚实、有能力、以股东利益为先？
- 业务是否简单到一个非专业人士也能理解？（他常说"不投自己看不懂的"）

## 2. 彼得·林奇视角：PEG Ratio 估值法

### 思想来源

彼得·林奇（Peter Lynch），麦哲伦基金传奇经理人，创造了13年29.2%年化收益率的纪录。他用 PEG Ratio 来衡量"你为增长付出的价格是否合理"。

### 核心公式

```
PEG Ratio = 市盈率（P/E） ÷ 盈利增长率（EPS Growth Rate %）
```

**通俗理解**：如果一家公司市盈率是30倍（你花30块买1块的利润），但利润每年增长30%，那PEG=1，说明你付出的价格和增长速度匹配。如果PEG=2，说明你为每单位增长付了双倍的钱——太贵了。

### 搜索和计算步骤

1. **搜索关键词**：`[股票代码] PE ratio forward` `[股票代码] EPS growth rate estimate`
2. **使用远期PE（Forward P/E）**：基于未来12个月预期盈利的PE，比历史PE更适合成长股
3. **盈利增长率**：优先使用分析师共识预期的未来3-5年年均EPS增长率
4. **计算 PEG**

### 判断标准（林奇原版 + 科技股调整）

| PEG Ratio | 判断 |
|-----------|------|
| < 0.5 | 极具吸引力（可能被严重低估，但要检查增长预期是否合理） |
| 0.5 - 1.0 | 有吸引力（增长与估值匹配良好） |
| 1.0 - 1.5 | 合理（对高质量科技公司可接受） |
| 1.5 - 2.0 | 偏贵（需要超强竞争优势支撑） |
| > 2.0 | 昂贵（除非有极特殊的增长催化剂） |

### 林奇会额外关注什么

- 他把股票分为6类：缓慢增长型、稳健增长型、快速增长型、周期型、资产型、困境反转型。科技股多属"快速增长型"，关键是增长是否可持续。
- 他喜欢"机构投资者关注度低"的股票（但大型科技股通常不符合这个条件）
- 他关注"故事"是否合理——公司靠什么赚钱？增长动力来自哪里？

## 3. 反向DCF视角：Reverse DCF

### 思想来源

传统DCF（折现现金流模型）是给一大堆未来假设然后算出一个"内在价值"。问题是：垃圾假设进去，垃圾估值出来（Garbage In, Garbage Out）。反向DCF的逻辑完全反过来——从当前股价出发，反推出"市场正在定价的隐含增长率是多少"，然后你判断这个预期合不合理。

### 核心逻辑

```
当前股价 → 反推 → 市场隐含的未来收入/盈利增长率
↓
你来判断：这个隐含增长率，合理吗？
```

**通俗理解**：就像买房。传统DCF是你估算这房子该值多少钱。反向DCF是看标价后问：以这个价格，这房子需要每年涨多少才值这个价？你觉得现实吗？

### 简化计算步骤

1. **获取数据**：
   - 当前市值（Market Cap）或企业价值（EV = 市值 + 净负债）
   - 当前自由现金流（FCF）或营业利润
   - 加权平均资本成本（WACC）：科技股通常用8-12%。搜索 `[股票代码] WACC` 或用经验值：大型科技8-10%，中型科技10-12%
   - 终值增长率（Terminal Growth Rate）：通常假设2-3%（长期GDP增速附近）

2. **反推隐含增长率**：
   ```
   当前EV = FCF × (1+g) / (WACC-g) × [增长期乘数]
   
   解方程求 g（隐含增长率）
   ```
   
   简化方法：如果搜索到专业来源已经计算了反向DCF，直接引用即可。搜索 `[股票代码] reverse DCF implied growth rate`

3. **判断隐含增长率是否合理**：
   - 与公司历史增长率对比
   - 与行业增长率对比
   - 与管理层指引对比
   - 与分析师预期对比

### 判断标准

| 隐含增长率 vs 合理预期 | 判断 |
|---------------------|------|
| 隐含增长率 < 历史增速 & 分析师预期 | 可能被低估（市场悲观过度） |
| 隐含增长率 ≈ 合理预期 | 合理定价 |
| 隐含增长率 > 预期20%以上 | 定价偏乐观，需要"超预期"表现才能支撑股价 |
| 隐含增长率远超行业天花板 | 严重高估，市场在做白日梦 |

## 4. 格林布拉特视角：魔法公式（Magic Formula）

### 思想来源

乔尔·格林布拉特（Joel Greenblatt），《股市稳赚》（The Little Book That Beats the Market）作者。他用两个简单指标的组合来找"又好又便宜"的股票。

### 核心公式

```
指标1：盈利收益率（Earnings Yield）= EBIT / 企业价值（EV）
→ 反映"便宜程度"——你每花1块钱企业价值能获得多少利润

指标2：投入资本回报率（ROIC）= EBIT / (净营运资本 + 净固定资产)
→ 反映"质量"——公司用投入的资本能产生多高的回报
```

**通俗理解**：就像买二手车。盈利收益率是看"性价比"——这车以这个价格划不划算。ROIC是看"车况"——这车本身性能好不好。最好的投资是"又便宜又好用"。

### 搜索和计算步骤

1. **搜索关键词**：`[股票代码] EBIT enterprise value` `[股票代码] ROIC return on invested capital`
2. **计算 Earnings Yield**：EBIT ÷ EV（EV = 市值 + 总负债 - 现金）
3. **获取 ROIC**：许多金融网站直接提供此数据
4. **综合判断**：两个指标分别排名，综合排名靠前 = 好标的

### 判断标准

| 维度 | 优秀 | 良好 | 一般 | 差 |
|------|------|------|------|-----|
| Earnings Yield | > 8% | 5-8% | 3-5% | < 3% |
| ROIC | > 25% | 15-25% | 10-15% | < 10% |

**组合判断**：
- Earnings Yield 高 + ROIC 高 = ⭐ 魔法公式首选（又好又便宜）
- Earnings Yield 低 + ROIC 高 = 好公司但贵了（等回调）
- Earnings Yield 高 + ROIC 低 = 便宜但质量一般（可能是价值陷阱）
- Earnings Yield 低 + ROIC 低 = 两个维度都不吸引

## 5. 达摩达兰视角：EV/EBITDA 行业对标

### 思想来源

阿斯瓦斯·达摩达兰（Aswath Damodaran），纽约大学教授，被称为"估值教父"。他强调估值必须放在行业背景下比较，而不是孤立地看一家公司。

### 核心指标

```
EV/EBITDA = 企业价值 ÷ 息税折旧摊销前利润
```

**为什么用EV/EBITDA而不是P/E？**
- EV（企业价值）考虑了负债，比单纯市值更全面
- EBITDA剔除了资本结构（利息）、税率差异、折旧政策差异的影响，让不同公司之间更具可比性

**通俗理解**：就像比较不同餐厅。有的餐厅自有物业（无负债），有的租房（有贷款）。如果只看老板口袋里有多少钱（净利润），不公平。EV/EBITDA相当于"不管你是自有还是租的，看你实际经营赚了多少"。

### 搜索和计算步骤

1. **搜索目标公司**：`[股票代码] EV/EBITDA`
2. **搜索行业中位数**：`[行业名] average EV/EBITDA multiple` 或参考达摩达兰每年更新的行业估值数据表（搜索 `Damodaran industry EV/EBITDA`）
3. **搜索可比公司**：选3-5家最直接的竞争对手，获取其EV/EBITDA
4. **对比分析**

### 科技子行业参考基准（会随市场变化，需搜索验证最新数据）

| 子行业 | 典型EV/EBITDA范围 | 说明 |
|--------|------------------|------|
| 大型SaaS/云 | 20-40x | 高增长、高经常性收入溢价 |
| 成熟软件 | 15-25x | 稳定现金流 |
| 半导体 | 12-20x | 周期性折价 |
| 消费互联网 | 15-30x | 取决于增长和货币化能力 |
| 硬件/设备 | 8-15x | 较低倍数 |

### 判断标准

| 相对行业中位数 | 判断 |
|-------------|------|
| 低于行业中位数20%+ | 可能被低估（需排除基本面恶化原因） |
| 在行业中位数附近（±10%） | 合理定价 |
| 高于行业中位数20-50% | 溢价定价（需要更优的增长或质量支撑） |
| 高于行业中位数50%+ | 高溢价（需要非常强的理由） |

## 6. 科技股专属视角：EV/Revenue + Rule of 40

### 思想来源

这不是某一位大师的方法，而是硅谷风投和成长型投资者广泛使用的框架。对于高增长但尚未稳定盈利的科技公司，传统的PE和EV/EBITDA可能不适用（因为利润为负或波动大），这时候看收入倍数更有意义。

### 核心指标

```
指标1：EV/Revenue（企业价值 / 收入）
→ 每1块钱收入被市场定价多少钱

指标2：Rule of 40 分数 = 收入增长率(%) + 利润率(%)
→ 衡量"增长 vs 盈利"的平衡。一家公司增长20%、利润率20% = 40分，及格。
   增长50%、亏损10% = 40分，也及格（高增长弥补了亏损）。
```

### 搜索和计算步骤

1. **EV/Revenue**：`[股票代码] EV/Revenue` 或 `[股票代码] price to sales enterprise value`
2. **Rule of 40**：
   - 收入增长率：最近一个季度的YoY增长率
   - 利润率：用FCF Margin或EBITDA Margin（对科技公司通常用FCF Margin更准确）
   - 两者相加

### EV/Revenue 判断基准

| EV/Revenue | 增长率 > 30% | 增长率 15-30% | 增长率 < 15% |
|-----------|-------------|-------------|-------------|
| < 5x | 极具吸引力 | 有吸引力 | 可能有原因的便宜 |
| 5-10x | 有吸引力 | 合理 | 偏贵 |
| 10-20x | 合理 | 偏贵 | 昂贵 |
| 20-40x | 可接受（高增长溢价） | 昂贵 | 极度高估 |
| > 40x | 需要极强叙事 | 泡沫风险 | 泡沫 |

### Rule of 40 判断标准

| 分数 | 判断 |
|------|------|
| > 60 | 🏆 顶级——增长和盈利双优 |
| 40-60 | ✅ 健康——平衡良好 |
| 25-40 | 🟡 及格线以下——需要在某个维度改善 |
| < 25 | ❌ 不及格——增长和盈利都有问题 |

### 特殊考量

- **AI相关公司的收入质量溢价**：如果收入中有大比例的AI相关新业务，市场可能给更高的EV/Revenue倍数，因为投资者在定价未来的增长潜力。但要区分"真AI收入"和"蹭AI概念"。
- **毛利率的重要性**：同样10亿收入，毛利率80%的SaaS公司和毛利率30%的硬件公司不可同日而语。EV/Gross Profit 比 EV/Revenue 更公平。搜索 `[股票代码] gross margin` 来调整判断。

## 敏感性分析（Sensitivity Analysis）—— 必须执行

**为什么必须做敏感性分析**：单点估值（"合理价格是$250"）创造了虚假的精确感。现实中，你的假设几乎一定有误差。敏感性分析展示"如果我的假设偏差了10-20%，估值会怎么变"，帮助你理解结果对哪个变量最敏感。

**执行方法**：为每个主要估值模型选择2个最关键的输入变量，构建二维矩阵。

## Action Price 推导（独立于当前股价）

**关键理念**：如果你的"买入价"是通过"当前股价打八折"得来的，你被锚定了。正确的做法是：

1. **完全不看当前股价**，独立运行你的估值模型
2. 得出一个"合理价值区间"
3. 在合理价值的基础上，要求15-20%的安全边际（Margin of Safety）
4. 安全边际之后的价格 = 你的 Action Price
5. **然后**才去看当前股价，比较Action Price和市场价

```
独立估值 → 合理价值 → 减去安全边际 → Action Price → 对比当前股价
```

## IRR 门槛检查（回报率门槛）

**IRR（Internal Rate of Return，内部收益率）**：简单理解就是"这笔投资的预期年化回报率"。

```
IRR ≈ (目标价 / 当前价格)^(1/年数) - 1
```

| 仓位类型 | 最低IRR门槛 | 理由 |
|---------|-----------|------|
| 做多（核心持仓） | ≥ 15% | 必须大幅跑赢指数才值得承担集中风险 |
| 做多（战术持仓） | ≥ 20% | 短期持仓机会成本更高 |
| 做空 | ≥ 20-25% | 补偿做空的无限下行风险、借券成本和时机风险 |

**铁律**："好公司+合理价格" = PASS，不是BUY。"好公司+有吸引力的价格+IRR超过门槛" = BUY。

如果计算出的预期IRR未达门槛，无论公司基本面多好、叙事多动人，推荐必须是PASS + 注明"在$X价格我们会行动"。

## 综合使用指南

分析完6个估值模型后，需要做以下汇总：

1. **投票机制**：6个模型中，有几个指向低估、几个指向合理、几个指向高估？
2. **权重调整**：
   - 已盈利的成熟科技公司：Owner Earnings和EV/EBITDA权重更高
   - 高增长未盈利公司：EV/Revenue + Rule of 40和反向DCF权重更高
   - 所有类型：行业对标都应作为校验
3. **矛盾处理**：如果不同模型给出截然不同的结论，这本身就是一个重要信号——说明市场定价可能处于"分歧区"，需要特别谨慎
4. **IRR门槛**：所有模型都过了之后，最终还要过IRR门槛这一关
5. **Action Price**：独立推导，不锚定当前股价

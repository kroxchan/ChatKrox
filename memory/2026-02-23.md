## Key learnings / patterns observed
- Windows 工具链排障优先做“可验证的最小修复”，并把副作用写清楚（尤其是 PATH 变更）。
- Cron/自动化会话上下文不稳定，必须把 SOP/关键状态落到文件（daily log + anchors）而不是依赖对话记忆。
- 运营类（XHS）如果同时存在多份内容日历/台账文件，会产生事实分叉；要先定义单一 source of truth，再谈自动化写入与状态流转。
- 复杂行政/维权问题的写作模式：先锁定可核验硬事实（法院调取/材料链条异常），再设计最小闭环（书面回复+期限）与升级梯度（平台留痕督办 -> 纪委/政府/省厅）。

## Mistakes / gotchas to avoid (reflect-learn)
- Windows shell: 在工具 `exec` 场景下不要用 PowerShell 的 `&&` 链式；用 `cmd /c` 或 PowerShell 的 `;`。
- PATH 修复: 直接把 `HKCU\\Environment\\Path` 重写成单一路径能救火但可能破坏其它工具链；应先导出现有 Path，做增量合并或按清单恢复。
- 天气/外部数据: fetch 成功但解析/字段缺失时，不要误报“不可用”；区分“请求失败”和“解析失败”，并保留原始证据便于追查。
- 会话盘点: `sessions_list` 可能不覆盖所有渠道/主会话；复盘时应以 `memory/YYYY-MM-DD.md` 与可用 transcript 为准。
